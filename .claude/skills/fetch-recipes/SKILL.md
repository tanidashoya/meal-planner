# 🍳 レシピ自動収集スキル（Bash 実行）

このスキルは **Claude Code の Bash ツールを使って**
ローカルの Python スクリプトを実行する。

Claude の内蔵 WebFetch / WebSearch は **使用しない**。

---

## 🎯 やること

ユーザーの自然言語指示から以下を決定し、  
**必ず Bash 経由で `fetch_recipes.py` を実行すること。**

- プラットフォーム
  - 現在は **ナディアのみ対応**（他サイトは将来拡張予定）
- カテゴリ
  - 肉料理 / 魚料理 / 麺料理 / 丼・ルー料理 / 汁物 / その他
- 検索クエリ（1 語または複数語）
- 取得件数
- 出力方法
  - Google Sheets 追記 または TSV 出力

---

## 🚀 実行ポリシー（重要）

- Claude Code は Bash 実行時、**必ず `--quiet` オプションを付与する**
- レシピの詳細データ（TSV / 表形式）を Claude に表示しない
- Claude は以下のみをユーザーに報告する：

  - 成功時：取得件数と処理結果（例：Sheets に追記完了）
  - 失敗時：処理失敗の事実のみ（詳細は内部ログ）

- データの正は **Google Sheets または出力ファイル**とする

## 📤 出力方法のデフォルトルール（必須）

本スキルでは、**Google Sheets への追記をデフォルトの出力先とする。**

### デフォルト動作

- Claude Code は特別な指示がない限り、
  **必ず Bash 実行時に `--sheets-append` を付与すること**
- TSV 出力は補助的な出力手段とし、
  ユーザーから明示的に指定された場合のみ使用する

### Sheets 追記時の必須オプション

Google Sheets に出力する場合、Claude Code は
**必ず以下のオプションを付与して Bash 実行を行うこと。**

```bash
--sheets-append
```

## 🐍 Python 実行環境について（重要）

本スキルでは、Python 実行時に **必ず `python3` コマンドを使用すること。**

- `python` コマンドは存在しない環境があるため使用してはいけない
- Claude Code は Bash 実行時、必ず `python3` を用いて実行する

## 🧭 自然言語 → Bash 引数 正規化ルール（最重要）

本スキルはユーザーの自然言語指示を受け取り、
**Bash 実行時には必ずスクリプト仕様に完全一致する引数へ変換する。**

Claude Code は「自然言語の揺れ」をそのまま Bash に渡してはならない。

---

### 🔧 プラットフォーム名の正規化ルール

ユーザーの指示に以下のような表現が含まれる場合：

- ナディア
- nadia
- Nadia
- oceans-nadia

Claude Code は **必ず以下の正式表記に正規化して実行すること**：

```bash
--platform ナディア
```

---

## 🧠 カテゴリ自動判定ルール（対応表なし）

ユーザーの指示にカテゴリ指定が含まれない場合、
Claude Code は以下の思考手順で `--category` を必ず決定する。

### 判定対象カテゴリ（固定）

- 肉料理
- 魚料理
- 麺料理
- 丼・ルー料理
- 汁物
- その他

### 判定アルゴリズム（必須）

1. 指示文中の **主語となる料理・食材・料理名**を抽出する
2. それが以下のどれに最も自然に分類されるかを判断する：

   - 主素材が魚介類 → 魚料理
   - 主素材が肉類 → 肉料理
   - 主食が麺類 → 麺料理
   - 丼・カレー・シチューなど一皿完結型 → 丼・ルー料理
   - 汁が主体の料理 → 汁物
   - 上記に明確に当てはまらない → その他

3. **迷った場合は必ず「その他」を選ぶ**
4. 判定結果は Bash 実行時の `--category` に必ず反映する

この判定は対応表・固定辞書を使わず、
**言語的意味理解に基づいて行う。**

---

## 🔍 検索クエリ（query）の保存について

本スキルでは、レシピ取得時に使用した **検索クエリ（query）を
各レシピデータに `search_query` として必ず保存する。**

## 🧾 レシピメタ情報の取得について

本スキルでは、各レシピページから以下の情報を
**スクレイピングによって取得し、構造化データとして保存する。**

- description
  - レシピページに記載された説明文（導入文・概要文）
- time
  - 調理時間（例：「10 分」「15〜20 分」などの表記を含む）

### 補足

- これらの情報は Claude によって生成・補完されるものではない
- 必ずレシピページ上の一次情報をスクレイピングして取得する
- TSV / Google Sheets の両方に含める

これにより、本スキルは
**レシピ本文・条件情報を含む正規データ生成スキル**
として機能する。

### 目的

- レシピそのものの内容とは別に
  **「どの検索意図で取得されたデータか」**を構造化して保持するため
- 後続の処理（AI 検索・再学習・分析）において、
  クエリ単位での評価・重み付け・再利用を可能にするため

### 補足

- `search_query` はレシピ説明文（description）の代替ではない
- 検索結果として取得された **文脈情報（メタデータ）**として扱う
- TSV / Google Sheets 出力の両方に必ず含める

この設計により、本スキルは
**「検索意図を含んだレシピデータ生成パイプライン」**として機能する。

---

## 🔁 既存レシピとの重複排除について

このスキルは、既存レシピが保存された CSV ファイルを
重複排除用の入力として受け取ることがある。

- Claude Code は CSV の中身を確認・解釈しない
- CSV はそのまま Bash 実行時の引数として渡す
- 重複排除の判定および処理は **すべて Python スクリプト側で行う**

Claude Code は、処理結果の成否のみをユーザーに報告する。

---

## 🔧 検索クエリ指定のルール（重要）

検索キーワードを指定する場合、
**必ず `--queries` 引数を使用すること。**

- 単一キーワードの場合でも `--queries` を使う
- `--query` という引数は **存在しないため使用してはいけない**
- Claude Code は検索語が 1 つの場合でも、
  **必ず `--queries` を使用する**

### 例

```bash
--queries パスタ
```

## ▶ 実行例（Google Sheets 追記・デフォルト）

以下は、本スキルの**標準的な実行例**である。

```bash
python3 fetch_recipes.py \
  --platform ナディア \
  --category 肉料理 \
  --count 1 \
  --queries 豚こま \
  --sheets-append \
  --quiet
```
